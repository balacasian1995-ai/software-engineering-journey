**What is the difference between Git and GitHub?**
Git is a version control system that runs locally on your computer and tracks changes to files over time. GitHub is a cloud-based platform that hosts Git repositories online, adding features like collaboration tools, pull requests, issue tracking, and remote backup. Git works without GitHub, but GitHub relies on Git.

**What is a repository?**
A repository (or "repo") is a folder tracked by Git that stores your project's files along with the full history of changes made to them.

**What is a commit?**
A commit is a saved snapshot of changes to files in a repository, along with a message describing what was changed and why.

**What is a branch?**
A branch is an independent line of development within a repository. It lets you work on changes (like a new feature or assignment) separately from the main codebase without affecting it until you're ready to merge.

**What is a pull request?**
A pull request (PR) is a request to merge changes from one branch into another (often into `main`). It allows others to review, comment on, and discuss the changes before they're merged.

**What is the difference between a Docker image and a Docker container?**
A Docker image is a static, read-only template that defines an application and its environment (code, dependencies, configuration). A Docker container is a running instance of that image — a live, isolated process created from the image.

**Why can Docker reduce "works on my machine" problems?**
Docker packages an application together with its exact dependencies, libraries, and configuration into a single portable image. Because the container runs the same way regardless of the host machine, it eliminates inconsistencies caused by differing environments, installed versions, or OS settings.
